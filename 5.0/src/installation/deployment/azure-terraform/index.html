Azure Terraform Deployment

Overview

Deploy XMPro on Azure using our production-ready Terraform module. This documentation provides implementation guidance, while technical specifications are maintained in the module's GitHub repository.

Note

For v4.4 and earlier, use the ARM template deployment. This Terraform module is for XMPro v4.5 and later.

Architecture

The Terraform module deploys a complete XMPro platform on Azure:

The module creates:

  • Core Applications: SM, AD, DS, and optionally AI services
  • Azure Infrastructure: SQL databases, App Services, Key Vault, Storage
  • Security: Managed identities, SSL certificates, network security
  • Monitoring: Application Insights and Log Analytics

For detailed component descriptions, resource dependencies, and technical specifications, see the Architecture Overview and Submodules sections in the GitHub README.

License Management

The Terraform module supports two licensing modes:

Evaluation Mode

When is_evaluation_mode = true, the deployment includes:

  • Automatic license provisioning via licenses container
  • Pre-configured evaluation product IDs and keys
  • No manual license requests required
  • Forces company name to "Evaluation" (overrides any company_name variable value)
Important

To use a custom company name, you must set is_evaluation_mode = false and request licenses from XMPro for your specific company name.

Production Mode (Default)

When is_evaluation_mode = false (default), you need to manually request licenses:

  1. Login using the Super Admin account admin@xmpro.onxmpro.com
  2. Navigate to Company â†’ Subscriptions
  3. Click "Update License" for each product (AD, DS)
  4. Generate license requests and submit to XMPro support
  5. Upload received licenses
Note

License requests require SMTP configuration to be enabled during deployment. See the Module Documentation for SMTP setup.

Upgrade

To upgrade an existing XMPro Terraform deployment to a newer version:

Warning

Before upgrading, back up your databases using Azure SQL Database export functionality.

Step 1: Prepare for Upgrade

# Navigate to your deployment directory
cd xmpro-deployment  # Your deployment directory

# Back up current state
cp terraform.tfstate terraform.tfstate.backup

Step 2: Update Configuration

Update your Terraform configuration to use the new version:

# Option A: Update imageversion variable
# Edit your terraform.tfvars file and update:
# imageversion = "4.5.2"  # Upgrade from 4.5.0 to 4.5.2

# Option B: Use command line override
terraform plan -var="imageversion=4.5.2"

Step 3: Plan Your Upgrade

terraform plan

Review the planned changes before proceeding.

Step 4: Apply Configuration

terraform apply

Step 5: Verify Deployment

After upgrade, verify your applications are running:

# List deployed resources
az resource list --resource-group $(terraform output -raw resource_group_name) --output table

# Check application versions
curl -f "$(terraform output -raw ad_app_url)/version"
curl -f "$(terraform output -raw ds_app_url)/version"
curl -f "$(terraform output -raw sm_app_url)/version"

Step 6: Access Applications

Navigate to the URLs provided by terraform output:

  • SM (Subscription Manager): Verify user access and licenses
  • AD (App Designer): Test application functionality
  • DS (Data Stream Designer): Verify data pipeline operations

Post-Upgrade Validation

  1. Test Core Functionality

    • Login with existing user accounts
    • Verify applications and data streams function correctly
    • Check that data is preserved and accessible

  2. Monitor Performance

    • Check Application Insights for any errors
    • Monitor resource utilization
    • Verify all services are responding normally

For upgrade troubleshooting, see the Troubleshooting Guide.

Uninstall

To completely remove the XMPro platform:

  1. Destroy Terraform resources:

    cd xmpro-deployment  # Your deployment directory
terraform destroy

  2. Manual cleanup (if needed):

    • Remove the resource group from Azure Portal
    • Remove Terraform state files (if using local state)
Warning

This will permanently delete all data. Ensure you have backups before proceeding.
