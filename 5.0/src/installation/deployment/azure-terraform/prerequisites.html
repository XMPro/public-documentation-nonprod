Prerequisites and Setup

Introduction

This guide covers the prerequisites and initial setup required for deploying XMPro on Azure using Terraform. Follow these steps carefully to ensure a successful deployment.

Software Installation

1. Install Terraform

Windows (using Chocolatey):

choco install terraform

macOS (using Homebrew):

brew install terraform

Linux (Ubuntu/Debian):

curl -fsSL https://apt.releases.hashicorp.com/gpg | sudo apt-key add -
sudo apt-add-repository "deb [arch=amd64] https://apt.releases.hashicorp.com $(lsb_release -cs) main"
sudo apt-get update && sudo apt-get install terraform

Manual Installation:

  1. Download from terraform.io/downloads
  2. Extract to a directory in your PATH
  3. Verify installation: terraform version

2. Install Azure CLI

Windows:

Invoke-WebRequest -Uri https://aka.ms/installazurecliwindows -OutFile .\AzureCLI.msi
Start-Process msiexec.exe -Wait -ArgumentList '/I AzureCLI.msi /quiet'

macOS:

brew install azure-cli

Linux:

curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

Alternative - Using pip:

pip install azure-cli

Azure Setup

1. Create Azure Service Principal

A service principal is required for Terraform to authenticate with Azure.

Using Azure CLI:

# Login to Azure
az login

# Set subscription
az account set --subscription "Your Subscription Name"

# Create service principal
az ad sp create-for-rbac --name "terraform-xmpro" --role="Contributor" --scopes="/subscriptions/YOUR_SUBSCRIPTION_ID"

Expected output:

{
  "appId": "12345678-1234-1234-1234-123456789012",
  "displayName": "terraform-xmpro",
  "password": "your-client-secret",
  "tenant": "87654321-4321-4321-4321-210987654321"
}

Save these values:

  • appId = ARM_CLIENT_ID
  • password = ARM_CLIENT_SECRET
  • tenant = ARM_TENANT_ID
  • Your subscription ID = ARM_SUBSCRIPTION_ID

2. Verify Service Principal

# Test service principal login
az login --service-principal \
  --username "your-app-id" \
  --password "your-client-secret" \
  --tenant "your-tenant-id"

# List subscriptions
az account list --output table

3. Configure Azure CLI

# Set default subscription
az account set --subscription "Your Subscription Name"

# Verify configuration
az account show

Container Registry Access

1. Azure Container Registry (ACR)

If using a private ACR, obtain credentials from your administrator:

# Login to ACR
az acr login --name xmprocontainers

# Test image access
az acr repository list --name xmprocontainers

Environment Configuration

1. Environment Variables

Create a .env file or set system environment variables:

# Azure Authentication
export ARM_CLIENT_ID="your-client-id"
export ARM_CLIENT_SECRET="your-client-secret"
export ARM_SUBSCRIPTION_ID="your-subscription-id"
export ARM_TENANT_ID="your-tenant-id"

# Container Registry
export TF_VAR_acr_username="your-acr-username"
export TF_VAR_acr_password="your-acr-password"

# Application Secrets
export TF_VAR_db_admin_password="SecurePassword123!"
export TF_VAR_company_admin_password="AdminPassword123!"
export TF_VAR_site_admin_password="SitePassword123!"
export TF_VAR_masterkey="YourMasterKey123!"

Windows PowerShell:

$env:ARM_CLIENT_ID="your-client-id"
$env:ARM_CLIENT_SECRET="your-client-secret"
$env:ARM_SUBSCRIPTION_ID="your-subscription-id"
$env:ARM_TENANT_ID="your-tenant-id"

2. Terraform Backend Configuration

For production deployments, configure remote state storage:

terraform {
  backend "azurerm" {
    resource_group_name  = "terraform-state-rg"
    storage_account_name = "terraformstatestore"
    container_name       = "terraform-state"
    key                  = "xmpro-dev.tfstate"
  }
}

Security Configuration

1. Password Requirements

Ensure all passwords meet Azure security requirements:

  • Minimum length: 12 characters
  • Complexity: Mix of uppercase, lowercase, numbers, symbols
  • Avoid: Dictionary words, personal information
  • Unique: Different passwords for each service

2. Key Vault Setup

For enhanced security, pre-create Key Vault resources:

# Create Key Vault
az keyvault create \
  --name "xmpro-dev-kv" \
  --resource-group "terraform-state-rg" \
  --location "East US"

# Set access policy
az keyvault set-policy \
  --name "xmpro-dev-kv" \
  --object-id "your-service-principal-object-id" \
  --secret-permissions get list set delete

Troubleshooting Setup Issues

Common Installation Problems

Terraform not found:

# Check PATH
echo $PATH

# Verify installation location
which terraform

# Reinstall if needed

Azure CLI authentication fails:

# Clear cached credentials
az logout
az login --use-device-code

# Check tenant/subscription
az account show

Permission errors:

# Check Azure permissions
az role assignment list --assignee "$ARM_CLIENT_ID"

# Verify subscription access
az account show --subscription "$ARM_SUBSCRIPTION_ID"

Environment Variable Issues

Variables not set:

# Check environment variables
printenv | grep ARM_
printenv | grep TF_VAR_

# Set missing variables
export ARM_CLIENT_ID="your-value"

Windows PowerShell variables:

# Check variables
Get-ChildItem Env:ARM_*
Get-ChildItem Env:TF_VAR_*

# Set variables
$env:ARM_CLIENT_ID="your-value"

Best Practices

1. Security

  • Never commit secrets to version control
  • Use Azure Key Vault for production secrets
  • Regularly rotate service principal credentials
  • Enable MFA for Azure account access

2. Development Workflow

  • Use separate environments for development/testing
  • Version control all Terraform configurations
  • Test changes in development before production
  • Document custom configurations

3. Maintenance

  • Keep Terraform and Azure CLI updated
  • Monitor Azure resource costs
  • Regular backup of Terraform state
  • Review and update security configurations
