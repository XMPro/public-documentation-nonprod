Prerequisites and Setup

Introduction

This guide covers the prerequisites and initial setup required for deploying XMPro on Azure using Terraform. Follow these steps carefully to ensure a successful deployment.

System Requirements

Software Installation

1. Install Terraform

Follow the official HashiCorp installation guide for your operating system:

Install Terraform - HashiCorp Developer

After installation, verify it works:

terraform version

2. Install Azure CLI

Windows:

Invoke-WebRequest -Uri https://aka.ms/installazurecliwindows -OutFile .\AzureCLI.msi
Start-Process msiexec.exe -Wait -ArgumentList '/I AzureCLI.msi /quiet'

macOS:

brew install azure-cli

Linux:

curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

Alternative - Using pip:

pip install azure-cli

3. Install Docker (Optional)

Docker is optional but recommended for testing container image access.

Windows:

  • Download and install Docker Desktop for Windows

macOS:

brew install --cask docker

Linux:

# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

XMPro Artifacts Access

1. XMPro Public Container Registry

XMPro container images are publicly available at {{ACR_URL}}. No authentication is required to pull these images.

# Test image access (no login required)
docker pull {{ACR_URL}}/ad:{{VERSION}}
docker pull {{ACR_URL}}/ds:{{VERSION}}
docker pull {{ACR_URL}}/stream-host:{{VERSION}}

2. XMPro Download Site

The SM (Subscription Manager) deployment package is downloaded from {{DOWNLOAD_URL}}. This site hosts the SM.zip file and other XMPro installation artifacts.

# The Terraform module automatically downloads SM.zip from:
# {{DOWNLOAD_URL}}/SM.zip
Note

SM (Subscription Manager) runs as a Windows App Service using a deployment package, not a container image.

Tip

If you're behind a corporate firewall, ensure that access to the following resources is allowed:

  • Container registries: {{ACR_URL}} and registry-1.docker.io (Docker Hub)
  • Download site: {{DOWNLOAD_URL}}

You may need to configure proxy settings or request firewall exceptions from your IT administrator.

Network Configuration

1. Firewall Rules

Ensure the following ports are accessible:

Service Port Protocol Purpose
HTTPS 443 TCP Web application access
HTTP 80 TCP HTTP redirect to HTTPS
SQL 1433 TCP Database connections (if needed)

2. DNS Configuration (Optional)

Custom domain configuration is optional. If not configured, the deployment will use the default *.azurewebsites.net domains provided by Azure App Service.

For custom domains, you'll need to manage DNS records after deployment. See the Module Documentation for details.

Validation Steps

1. Verify Tool Installation

# Check Terraform version
terraform version

# Check Azure CLI version
az version

# Check Docker version (if installed)
docker --version

2. Test Azure Authentication

Follow the verification steps in the Microsoft Azure Terraform guide to ensure your authentication is working correctly.

3. Test XMPro Artifacts Access

# No login required for public registry
# Test image pull directly
docker pull {{ACR_URL}}/ad:{{VERSION}}
docker pull {{ACR_URL}}/ds:{{VERSION}}
docker pull {{ACR_URL}}/stream-host:{{VERSION}}

# Verify images
docker images | grep xmpro

Troubleshooting Setup Issues

Common Installation Problems

Terraform not found:

# Check PATH
echo $PATH

# Verify installation location
which terraform

# Reinstall if needed

Azure CLI authentication fails:

# Clear cached credentials
az logout
az login --use-device-code

# Check tenant/subscription
az account show

Permission errors: Refer to the Microsoft Azure Terraform guide for troubleshooting authentication and permission issues.

Best Practices

1. Security

  • Never commit secrets to version control
  • Use Azure Key Vault for production secrets
  • Regularly rotate service principal credentials
  • Enable MFA for Azure account access

2. Development Workflow

  • Use separate environments for development/testing
  • Version control all Terraform configurations
  • Test changes in development before production
  • Document custom configurations

3. Maintenance

  • Keep Terraform and Azure CLI updated
  • Monitor Azure resource costs
  • Regular backup of Terraform state
  • Review and update security configurations
