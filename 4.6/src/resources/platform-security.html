Platform Security

XMPro places a high priority on security, performing app security checks every 3 months using Veracode. Veracode's comprehensive analysis helps identify, prevent, and fix vulnerabilities through multiple testing methods: Static Analysis (white-box testing), Dynamic Analysis (black-box testing), and Software Composition Analysis.

Security Technologies and Practices

Our suite of products leverages robust technologies and practices to maintain a high security standard:

  • App Designer, Data Stream Designer, and XMPro AI are built on dotnet 8.
  • Subscription Manager is built on Microsoft .NET Framework 4.8 Runtime.

In the event a vulnerability is identified in any of these technologies, Microsoft promptly releases an update. We integrate these updates into our products and regularly release new versions that include essential security fixes.

Static Application Security Testing (SAST)

Static Application Security Testing (SAST) is a form of white-box testing used to scan an application's source, binary, or byte code.

Dynamic Application Security Testing (DAST)

Dynamic Application Security Testing (DAST) analyzes a web application through the front end to find vulnerabilities through simulated attacks. This is also called Penetration testing.

Software Composition Analysis (SCA)

Software Composition Analysis scans all the components used in an application for security risks and vulnerabilities.

Container Security Scanning

In addition to application-level security testing, XMPro performs container image vulnerability scanning on every release using Trivy, an open-source vulnerability scanner. All container images published to the XMPro Azure Container Registry are scanned for known vulnerabilities in OS packages and application dependencies.

Each scan report includes:

  • Vulnerability counts by severity (Critical, High, Medium, Low)
  • False positive analysis for all Critical and High findings
  • Affected packages and CVE identifiers
  • Replication commands for independent verification

Known False Positives

The following CVEs have been assessed as false positives across XMPro container images. The affected components are either not present in XMPro deployments, not reachable in the container runtime environment, or are disputed by upstream maintainers. This list is cumulative and updated as new CVEs are investigated.

  • CVE-2005-2541: tar — Setuid extraction; intended POSIX behavior, won't fix.
  • CVE-2010-4756: glibc — Won't fix: glob resource consumption is POSIX-specified behavior.
  • CVE-2015-3276: openldap — TLS cipher acceptance; XMPro uses .NET LDAP, not libldap.
  • CVE-2016-2781: coreutils — chroot TIOCSTI escape; kernel mitigated since Linux 6.4.4.
  • CVE-2017-17740: openldap — contrib/slapd-modules/nops DoS; Debian doesn't build contrib modules.
  • CVE-2017-18018: coreutils — chown race condition; kernel hardening (fs.protected_regular) mitigates.
  • CVE-2018-5709: krb5 — Integer overflow in policy lifetime; trusted admin input only.
  • CVE-2018-6829: libgcrypt20 — ElGamal side-channel; .NET uses OpenSSL, hybrid mode unaffected.
  • CVE-2018-20225: pip < 19.2 — Vulnerable --extra-index-url option never used.
  • CVE-2018-20796: glibc — Disputed regex recursion DoS; glibc maintainers do not consider this a security issue.
  • CVE-2019-1010022: libc-bin (glibc) — Disputed CVE; glibc upstream states "not a real threat"; requires prior stack overflow.
  • CVE-2019-1010023: glibc — Disputed: ldd requires user to run on malicious ELF; documented behavior.
  • CVE-2019-1010024: glibc — Disputed ASLR bypass via exception messages; kernel security feature.
  • CVE-2019-9192: glibc — Disputed regex recursion DoS; same as CVE-2018-20796.
  • CVE-2022-0563: util-linux — chfn/chsh readline echo; Debian uses shadow package instead.
  • CVE-2022-27943: gcc — libiberty Rust demangling; dev tools only, negligible impact.
  • CVE-2023-2953: libldap-2.5-0 — No LDAP functionality used in Stream Host containers.
  • CVE-2023-31486: perl — HTTP::Tiny TLS verification; XMPro uses .NET HttpClient.
  • CVE-2023-42465: sudo — ROWHAMMER hardware attack; impractical in cloud with ECC RAM.
  • CVE-2024-1013: Alpine unixodbc — Example code only, not compiled.
  • CVE-2025-6297: dpkg — Only affects manual dpkg-deb extraction, not used in containers.
  • CVE-2025-7458: libsqlite3-0 — Python variant only; requires crafted SQL with large ORDER BY, not possible in containers.
  • CVE-2025-10148: curl — WebSocket payload masking; XMPro uses .NET HttpClient.
  • CVE-2025-13034: curl — QUIC TLS key pinning; XMPro uses .NET HttpClient.
  • CVE-2025-13151: libtasn1-6 — .NET uses OpenSSL not GnuTLS; vulnerable code never executed; no base image fix available.
  • CVE-2025-14104: util-linux — setpwnam race condition; SUID utilities not used in containers.
  • CVE-2025-14524: curl — OAuth2 Bearer token leak; XMPro uses .NET HttpClient.
  • CVE-2025-14819: curl — CA certificate store caching; XMPro uses .NET TLS.
  • CVE-2025-15281: libc-bin (glibc) — Requires wordexp() with WRDE_REUSE+WRDE_APPEND flags; not used by .NET applications.
  • CVE-2025-30258: gpgv — Certificate import DoS; XMPro doesn't import GnuPG keys.
  • CVE-2025-68972: gpgv — Plaintext injection in signed messages; XMPro doesn't verify signatures.
  • CVE-2026-0861: glibc — Requires attacker control of both size and alignment parameters; not possible in .NET managed code.
  • CVE-2026-2219: dpkg — zstd decompression infinite loop; only affects processing untrusted .deb archives.
  • CVE-2026-3805: curl / libcurl4t64 — Debian Trixie curl vulnerability; XMPro uses .NET HttpClient, not curl.
  • CVE-2026-22184: zlib (npm) — untgz utility not in browserify-zlib/minizlib; pure JS packages.
  • CVE-2026-24882: gpgv — Vulnerability in tpm2daemon not included in gpgv package; only signature verification tool.
