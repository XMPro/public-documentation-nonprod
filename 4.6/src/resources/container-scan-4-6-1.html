Container Security Scan Report - XMPro 4.6.1

Registry: xmpro.azurecr.io Version: 4.6.1 Release Date: 13 May 2026

Executive Summary

This report documents all container CVEs that have been investigated and confirmed as false positives at the 4.6.1 release. Each entry below identifies the affected component and the reason XMPro is not exposed. Eighteen new false positives were assessed since the 4.6.0 release; the remaining entries carry forward from earlier releases.

Status Count
Total documented false positives at 4.6.1 54
New since 4.6.0 18
Carried forward from 4.6.0 36
Note

A CVE listed here means a security scanner has flagged the vulnerability in an XMPro container image, but engineering has reviewed the issue and confirmed it does not affect XMPro at runtime. Common reasons include: the vulnerable code path is never executed, the affected library is transitively installed but not invoked, or the vulnerability requires preconditions that do not exist in XMPro deployments. Genuine vulnerabilities (true positives) are addressed in the release itself and do not appear on this page.

False Positive Assessments

Related: v4.6.1 Release Notes

The following CVEs were assessed as false positives. The affected components are either not present in XMPro deployments or are not reachable in the container runtime environment.

  • CVE-2005-2541: tar — Setuid extraction; intended POSIX behavior, won't fix.
  • CVE-2010-4756: glibc — Won't fix: glob resource consumption is POSIX-specified behavior.
  • CVE-2015-3276: openldap — TLS cipher acceptance; XMPro uses .NET LDAP, not libldap.
  • CVE-2016-2781: coreutils — chroot TIOCSTI escape; kernel mitigated since Linux 6.4.4.
  • CVE-2017-17740: openldap — contrib/slapd-modules/nops DoS; Debian doesn't build contrib modules.
  • CVE-2017-18018: coreutils — chown race condition; kernel hardening (fs.protected_regular) mitigates.
  • CVE-2018-5709: krb5 — Integer overflow in policy lifetime; trusted admin input only.
  • CVE-2018-6829: libgcrypt20 — ElGamal side-channel; .NET uses OpenSSL, hybrid mode unaffected.
  • CVE-2018-20225: pip < 19.2 — Vulnerable --extra-index-url option never used.
  • CVE-2018-20796: glibc — Disputed regex recursion DoS; glibc maintainers do not consider this a security issue.
  • CVE-2019-1010022: libc-bin (glibc) — Disputed CVE; glibc upstream states "not a real threat"; requires prior stack overflow.
  • CVE-2019-1010023: glibc — Disputed: ldd requires user to run on malicious ELF; documented behavior.
  • CVE-2019-1010024: glibc — Disputed ASLR bypass via exception messages; kernel security feature.
  • CVE-2019-9192: glibc — Disputed regex recursion DoS; same as CVE-2018-20796.
  • CVE-2022-0563: util-linux — chfn/chsh readline echo; Debian uses shadow package instead.
  • CVE-2022-27943: gcc — libiberty Rust demangling; dev tools only, negligible impact.
  • CVE-2023-2953: libldap-2.5-0 — No LDAP functionality used in Stream Host containers.
  • CVE-2023-31486: perl — HTTP::Tiny TLS verification; XMPro uses .NET HttpClient.
  • CVE-2023-42465: sudo — ROWHAMMER hardware attack; impractical in cloud with ECC RAM.
  • CVE-2024-1013: Alpine unixodbc — Example code only, not compiled.
  • CVE-2025-6297: dpkg — Only affects manual dpkg-deb extraction, not used in containers.
  • CVE-2025-7458: libsqlite3-0 — Python variant only; requires crafted SQL with large ORDER BY, not possible in containers.
  • CVE-2025-10148: curl — WebSocket payload masking; XMPro uses .NET HttpClient.
  • CVE-2025-13034: curl — QUIC TLS key pinning; XMPro uses .NET HttpClient.
  • CVE-2025-13151: libtasn1-6 — .NET uses OpenSSL not GnuTLS; vulnerable code never executed; no base image fix available.
  • CVE-2025-14104: util-linux — setpwnam race condition; SUID utilities not used in containers.
  • CVE-2025-14524: curl — OAuth2 Bearer token leak; XMPro uses .NET HttpClient.
  • CVE-2025-14819: curl — CA certificate store caching; XMPro uses .NET TLS.
  • CVE-2025-15281: libc-bin (glibc) — Requires wordexp() with WRDE_REUSE+WRDE_APPEND flags; not used by .NET applications.
  • CVE-2025-30258: gpgv — Certificate import DoS; XMPro doesn't import GnuPG keys.
  • CVE-2025-68972: gpgv — Plaintext injection in signed messages; XMPro doesn't verify signatures.
  • CVE-2026-0861: glibc — Requires attacker control of both size and alignment parameters; not possible in .NET managed code.
  • CVE-2026-2219: dpkg — zstd decompression infinite loop; only affects processing untrusted .deb archives.
  • CVE-2026-3805: curl / libcurl4t64 — Debian Trixie curl vulnerability; XMPro uses .NET HttpClient, not curl.
  • CVE-2026-22184: zlib (npm) — untgz utility not in browserify-zlib/minizlib; pure JS packages.
  • CVE-2026-24882: gpgv — Vulnerability in tpm2daemon not included in gpgv package; only signature verification tool.

New in 4.6.1

  • CVE-2026-3833: GnuTLS — Case-sensitive nameConstraints comparison violating RFC 5280 §7.2; XMPro uses .NET TLS (OpenSSL-backed), no libgnutls PInvoke; libgnutls30 transitive via curl, build-time only.

  • CVE-2026-4878: libcap2 — TOCTOU race in cap_set_file() requires local attacker + privileged setcap caller; XMPro containers never invoke setcap/getcap and run unprivileged.
  • CVE-2026-5435: glibc — OOB write in deprecated DNS-record-printing helpers (TSIG); .NET uses managed networking via getaddrinfo, not libresolv print API.
  • CVE-2026-5450: glibc — scanf %mc 1-byte heap overflow with width > 1024; .NET applications use managed parsing APIs, not glibc scanf.
  • CVE-2026-5928: glibc — ungetwc buffer under-read on wide-char streams; .NET uses managed StreamReader / UTF-8, not glibc wide-character stdio.
  • CVE-2026-7598: libssh2 — Integer overflow in userauth_password; XMPro has no SSH/SCP/SFTP usage; libssh2 transitive via curl, build-time only.
  • CVE-2026-28388: OpenSSL — NULL deref in delta CRL processing; XMPro does not process CRLs; OpenSSL rates as Low.
  • CVE-2026-28389: OpenSSL — NULL deref in CMS EnvelopedData KeyAgreeRecipientInfo; XMPro does not process CMS; OpenSSL rates as Low.
  • CVE-2026-28390: OpenSSL — NULL deref in CMS EnvelopedData KeyTransportRecipientInfo (RSA-OAEP); XMPro does not process CMS; OpenSSL rates as Low.
  • CVE-2026-31790: OpenSSL — Uninitialized memory in RSASVE key encapsulation; XMPro does not use RSASVE; OpenSSL rates as Moderate.
  • CVE-2026-33845: GnuTLS — Integer underflow in DTLS handshake fragment reassembly; XMPro uses .NET TLS (OpenSSL-backed), no DTLS endpoints, no gnutls PInvoke.
  • CVE-2026-33846: GnuTLS — Heap buffer overflow in DTLS handshake fragment reassembly (sibling of CVE-2026-33845); same reasoning.
  • CVE-2026-35535: sudo — Privilege escalation via mailer privilege drop failure; sudo mailer not configured in XMPro containers; sudo used only for NOPASSWD operations.
  • CVE-2026-40200: musl libc — Stack corruption in qsort with >7M elements on 32-bit; XMPro is 64-bit and .NET uses managed sort.
  • CVE-2026-42010: GnuTLS — RSA-PSK auth bypass via NUL bytes in usernames; XMPro doesn't use RSA-PSK ciphersuites or libgnutls.
  • CVE-2026-42011: GnuTLS — nameConstraints chain-walking flaw; XMPro uses .NET TLS (OpenSSL chain validation), no libgnutls PInvoke.
