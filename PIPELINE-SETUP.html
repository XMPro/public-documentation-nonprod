Pipeline Setup Guide

Prerequisites

1. GitHub Personal Access Token (PAT)

You need a GitHub PAT to allow Azure DevOps to push to GitHub repositories.

Steps to create:

  1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Click "Generate new token (classic)"
  3. Give it a descriptive name: "Azure DevOps - XMPro Docs Deployment"
  4. Select scopes:
    • repo (Full control of private repositories)
  5. Generate token and copy it immediately (you won't see it again)

2. GitHub Repositories

Create these repositories in the XMPro organization:

  • xmpro/public-documentation - Production documentation
  • xmpro/public-documentation-nonprod - Non-production documentation

Repository settings:

  1. Make them public repositories
  2. Don't initialize with README (keep them empty)
  3. After first deployment, enable GitHub Pages:
    • Go to Settings → Pages
    • Source: Deploy from a branch
    • Branch: main / (root)
    • Save

Azure DevOps Pipeline Setup

1. Add GitHub Token to Pipeline

For each pipeline you create:

  1. Edit pipeline → Variables
  2. Add new variable:
    • Name: DOCS_DEPLOY_PAT_CLASSIC
    • Value: [Your GitHub PAT]
    • Keep this value secret: ✓ (checked)
  3. Save

Note: If this variable already exists in your pipeline, you can skip this step.

2. Create Pipelines

Create these pipelines in Azure DevOps:

Pipeline Name YAML File Purpose
Docs - Main Branch azure-pipelines.yml Deploys main branch to v4.4
Docs - Release 4.4 azure-pipelines-release-4.4.yml Deploys release/4.4 branch
Docs - Release 4.5 azure-pipelines-release-4.5.yml Deploys release/4.5 branch

For each pipeline:

  1. New pipeline → Azure Repos Git
  2. Select your repository
  3. Select "Existing Azure Pipelines YAML file"
  4. Branch: 20290-migrate-public-docs (for testing)
  5. Path: /public-documentation/[yaml-file-name]
  6. Save (don't run yet)
  7. Edit → Variables → Add GITHUB_TOKEN
  8. Save and run

3. First Run

The first run will:

  1. Build documentation
  2. Deploy to nonprod automatically
  3. Wait for manual approval (24 hour timeout)
  4. Deploy to production after approval

Testing Checklist

  • [ ] GitHub PAT created with repo scope
  • [ ] Both GitHub repos created (public-documentation and public-documentation-nonprod)
  • [ ] DOCS_DEPLOY_PAT_CLASSIC added to each pipeline as secret variable (or already exists)
  • [ ] Pipeline runs successfully and deploys to nonprod
  • [ ] Can access nonprod site at: https://xmpro.github.io/public-documentation-nonprod/
  • [ ] Manual approval works
  • [ ] Production deployment succeeds
  • [ ] Can access production site at: https://xmpro.github.io/public-documentation/

Troubleshooting

Authentication Errors

  • Error: fatal: could not read Password for 'https://***@github.com'
  • Solution: Ensure DOCS_DEPLOY_PAT_CLASSIC is set as a pipeline variable

Permission Errors

  • Error: Permission denied to push to repository
  • Solution: Ensure your GitHub PAT has 'repo' scope and hasn't expired

Empty Repository

  • Error: fatal: couldn't find remote ref main
  • Solution: This is expected on first run. The pipeline will create the main branch.

After Testing

Before merging the PR:

  1. Remove feature branch from pipeline triggers (search for TODO comments)
  2. Delete test pipelines if you created any
  3. Update pipeline names to remove "Test" or similar prefixes
  4. Enable GitHub Pages on both repositories
  5. Configure custom domains (when ready)
